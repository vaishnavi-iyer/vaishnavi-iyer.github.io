What a responsive site is, and why responsiveness is important.?

These days, people use internet and access websites on devices of all sizes and forms, now this is obviously a benefit for the users. However, this is not particularly a boon for the developer.

Imagine the no. of sizes that a possible display can be of and imagine coding for all devices individually and maintaining it. If you add one feature on a page then how many countless device, you will need to duplicate the code.

This problem can be solved by something called responsive design. Responsive design is a design fundamental that is based on the idea that the design itself should incorporate the needs of devices of different sizes. And this can be done by using the same code with a little device specific tweaks here and there.

Having a responsive design will save a lot of time, effort and money in writing the same code again and again. Also, it will save making repetitive errors and the countless man hours spent in correcting those errors

What mobile first design is, and why it's important.

Making a design as per large size page, makes it hard to rescale it for it to be able to into a small screen. However, it is comparatively easy to rescale size to fit a larger screen.

Think of an analogy, if you have a small house, when you move into a bigger house, you are able to fit all your stuff very easily and have room for more new things. However, if you are moving from a big house to a smaller house then all your stuff will be still packed up in boxes and will be all over the place and sometimes you might even have to get rid of a lot of stuff. It’s the same with designing a page… a room is the screen size and all your stuff are various elements on the screen.

Hence if you start with the small house, relocating is easy and similarly starting with designing for a small display and moving up from there is comparatively easy.

What frameworks are, and their pros and cons?

Frameworks are a set of code templates. Some frequently used bits of code is put into the framework so that the developer doesn’t have to redo the whole thing again and again.

Consider the analogy, you are trying to make a burrito at home and you get the readymade tortilla and pre-cut vegies, some canned beans and salsa from the super market and all you have to do is assemble it at home for a yummy burrito.

Framework is all the ready-made stuff and the yummy burrito is the awesome webpage. As a good designer all you have to do is to use the frameworks properly.

This has a number of advantages:

  • It’s easier to make the website when you have a few bits ready and tried and tested.
  • It is faster.
  • Clean and tidy code.
  • Can teach you some practices and coding conventions.
  • Browser compatibility and known problems are already addressed Helpful if many people are working on same code.

And everything has a con…

  • Is you don’t know the framework properly, then it can take a while to understand and then use it.
  • Code is not optimised. Since we only use the bits that we need, lot of other code is unused.
  • Slower learning curve as you are only actually using stuff and not doing much by yourself.

What a wireframe is and why we use it?

Wireframes are basically mock-ups of webpage that can be on a piece of paper or any other graphics tool.

It is not practical to directly code whatever comes into our minds. A good idea is to first create drawings of how you want your page to be and then make a design and then start coding. It’s important to plan our development activities so that we don’t end up doing re-work and wasting time.
