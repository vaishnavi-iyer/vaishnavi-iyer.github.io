JavaScript - some more of it
Date: 15th May 2016

My dear friend why do you always keep asking me these questions. Now that you have asked be prepared to get a little confused ;)

Every JavaScript statement is composed of certain variable, strings and weird symbols just like how English uses words and punctuations. Just like any other language, you need to practise it to master it.

A few of these are:

Parentheses ()

it is basically used only for two things, the first is to group a set of instructions typically mathematical and the second it contains the function arguments. Hmm now you must be thinking why group things, I’ll give you an e.g.

>

Try calculating 3-2*4+5 the answer would most certainly evaluate to 0 but you wanted the subtraction to take place first so what can you do, use parenthesis, just like in maths (3-2)*4+5 and this would evaluate to 9.

This is where we do grouping.

Now if you are wondering what a function is, go back to my JavaScript Basics blog, I most certainly remember explaining it to you. So the parenthesis is used to contain the arguments that are passed into a function.

So if you have two numbers, and you need to find which one is greater and which one is smaller you can do it using functions, the function will do that for you but how does it know which numbers to compare. These numbers need to be given to the function and while giving those numbers, you contain it in a parenthesis and give it to the function.

function findSmallerNumber (a,b) = {
if (a === b)
{
console.log(“ Numbers are equal”);
}
else if (a >b)
{
console.log(b + “is smaller”);
}
else
{
console.log(a + “is smaller”);
}
To invoke this function, the function is called as
findSmallerNumber (5, 7)
This will compare the numbers and the result will be “5 is smaller”

brackets []

The only places I think we use the brackets are is to denote arrays and while accessing variables from an array or an object.

The array and object definition is something I have explained to you in the JavaScript Basics blog. Defining an array.

var a = [“0”, “1”, “2”, “3”]

Now if I need to access the variable “3” I can access it as a[3] and it is similar for accessing the objects.

braces {}

braces are used to group multiple statements together to define an execution sequence. Like multiple statements in a loop or a function are defined in the brace.

One more important use of brace is to define objects

e.g.

var contact = {
“name1” : “contact1”,
“name2” : “contact2”,
“name3” : “contact3”
}

Single ' ' and Double Quotes " "

More often than not you can see yourself using the quotes interchangeably. They are basically used to contain strings. But there is one ground rule that while defining a string you need to be consistent with the quotes at the start and end of the string. They CANNOT be different.

Also, if you are planning to use a certain quote as part of the string like if you are quoting someone literally and want to use the double quotes then it is a good idea to use the single quotes while defining the string or if you plan to use apostrophe in your string try to use double quotes while defining it.

Also, these quotes are also used while defining objects and access object variables and that is when you need to remember to use double quotes. Most JSON libraries requires you to use double quotes while defining objects and it’s good to get into a practise of that.

e.g.

var contact = {
“name1” : “contact1”,
“name2” : “contact2”,
“name3” : “contact3”
}
