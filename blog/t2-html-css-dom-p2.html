What is grid based design?

Grids are structure imposed on a mock-up layout/ design. They offer an effective design approach for site layouts and assist in communicating site’s main messages clearly to the end user. They help in creating an ordered hierarchy, proportional relationships, and clear visual paths for the eye to travel.

They main use or advantage that grid has is that, it makes it easy for developers to translate the mock-ups into a semantic code.

The Basic grid layout can be created and saved as a CSS framework file and that can be reused for creating codes in future.

Grids also enable us to visualise the size of each part of a webpage.

What is all the hype about responsive webpage?

Responsive web design (RWD) is basically a methodology to design websites keeping in mind optimal viewing and interaction experience—easy reading and navigation with a minimum of resizing, panning, and scrolling—across a wide range of devices (from desktop computer monitors to mobile phones).

A website designed, considering the Responsive design principles, easily adapts the layout to the viewing environment, the type and size of the display, the viewing orientation etc.

This can be done by using different concepts:
a. Media Queries: they allow the page to use different css style rules based on the characteristics of the device.
b. The fluid grid concept: Defines the pages in terms of relative units like percentage instead of absolute units like pixels.
c. The 12 column grid layout: this divides the display into 12 columns based on percentage. This grid creates visual consistency between layouts, while allowing flexibility across a wide variety of designs. Responsive web design is becoming more important as the amount of mobile traffic now accounts for more than half of total internet traffic.

What is semantic structure?

A correctly defined html structure consists of html tags, head, body, and div tags. However, this tags have generic meaning and one can identify its design only by looking at the style sheet.

Having tags that have a generic meaning irrespective of the type of webpage are very handy and help developers visualize the page easily. These tags are called semantic tags. Some examples of these tags are H1, H2, H3, H4, image and p (paragraph) tags. If there is an image tag on the page then one can easily expect that there will be an image on the page.
