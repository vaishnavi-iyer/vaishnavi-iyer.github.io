Comparing JavaScript to HTML and CSS

A good and beautifully designed webpage is a coherent combination of three main elements:
1. HTML
2. CSS and
3. JavaScript

These elements work together with each other to give us our dream webpage. It’s a balanced combination of “Structure, Style and Interactivity”.

HTML: Marks the content and is the lays building blocks to structure your webpage. Elements like the title, paragraphs and images etc are part of this.

CSS: It is used to stylise your webpage, it adds expression to the simple text. It dictates as to how your webpage will look like.

JavaScript: It tells the browser how it should respond to events. Events include clicks, changing values in form, zoom in & out etc.

The browser understand these three languages well and will load the webpages accordingly. These three work together in sync however, its best practise to keep them separate as it makes the code simpler and easy to understand and also neat.

While HTML and CSS are static parts of the webpage... JavaScript adds dynamic elements to it.

Control flow and loops.

Computers are a really powerful being. But the only problem (actually kinda good thing) is that ‘THEY DO ONLY WHAT THEY ARE TOLD TO DO’.

Human beings are super power in the sense that they can tell the computers what to do ( if only it were that simple).

I need to tell the computer that please brush my teeth every day once in the morning after I get up (don’t dare wake me up). Simple.

But the computer doesn’t understand my words and I need to explain it. So I use the computers language.

Our options are: 1. For Loop, 2. While Loop, 3. do-while Loop 4. If/Else statements and 5 Switch Case.

Loops will basically make the computer check the condition multiple times and control is to determine when to come out of the loop. All of these can be used to do anything. Depends on how comfortable you are with the syntax. Like I said in the in the example you have to be explicit in your instructions otherwise the loops might run forever and can crash your computer. And to not let it crash we use the control statements.

Control flow is in itself a bigger concept which means to change control from place to another – it defines how the code will be executed. Like for example. The difference between accessing data from arrays and object literals.

Array is like a folder-file that you have and you can store anything you want in there. You know that your bills are in the first folder, your letters in the second and so on.

Object Literals on the other hand is a drawer it can have a small box having your earrings, a big box having your necklace and a file having your documents and your wallet with you money & cards.

To tell someone to get my bills from the folder-file, I tell them can you please get stuff from the folder 1 of the folder file. And to tell someone to get my earrings, I need to say that please get the earrings from the small box in the drawer.

Now you might think it’s easier to say get from 1 instead of saying get from the small box. Well kind of true but it depends on how much information you have to deal with.

For e.g. If the folder-file has hundred folder, it’s hard to remember what is there in which folder and saying box and wallet seems simpler but for a smaller set of logically simpler data, addressing it with numbers will be simpler and hence both have their advantages and disadvantages and should be user appropriately.

Explain what functions are and why they are useful.

A function is basically a group of statements that together perform a task. This is a very important aspect of coding as it can save us a lot of time and energy when the tasks are repetitive.

For example, if you are writing a program to sort a piece of information in a file. It’s simple you can write that logic but imagine your boss says you have to do this for all the files in your office. Now this is hard, instead I will just make a function a pass a parameter which in this case would be file name. Now whenever I have to sort the files, I just have to call the function and tell the file name.
